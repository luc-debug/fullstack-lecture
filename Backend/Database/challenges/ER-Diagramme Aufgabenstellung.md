# Aufgabe

1. Identifizieren Sie die Entitäten.
2. Bestimmen Sie die Attribute.
3. Legen Sie Primärschlüssel fest.
4. Modellieren Sie die Beziehungen und Kardinalitäten.
5. Zeichnen Sie das ER-Diagramm.

---

## Aufgabe 1 – Krankenhaus (leicht)

Ein Krankenhaus verwaltet Patienten und Ärzte.

- Jeder Patient besitzt eine Patienten-ID, Name und Geburtsdatum.
- Ärzte haben eine Arzt-ID, Name und Fachrichtung.
- Patienten können von mehreren Ärzten behandelt werden.
- Ärzte behandeln mehrere Patienten.
- Zu jeder Behandlung werden Datum und Diagnose gespeichert.

**Aufgabe:**
Modellieren Sie das Szenario als ER-Diagramm.

**Hinweis:**
Überlegen Sie, ob „Behandlung" eine eigene Entität sein sollte.

---

## Aufgabe 2 – Online-Shop (leicht bis mittel)

Ein Online-Shop verkauft Produkte.

- Jedes Produkt hat eine Produktnummer, einen Namen und einen Preis.
- Kunden besitzen eine Kundennummer, einen Namen und eine E-Mail-Adresse.
- Kunden können Bestellungen aufgeben.
- Eine Bestellung enthält mehrere Produkte.

---

## Aufgabe 3 – Universität (mittel)

Eine Universität verwaltet Studierende, Dozenten und Lehrveranstaltungen.

- Studierende können mehrere Lehrveranstaltungen belegen.
- Eine Lehrveranstaltung kann von vielen Studierenden besucht werden.
- Jede Lehrveranstaltung wird von genau einem Dozenten gehalten.
- Zu jedem Studierenden werden Matrikelnummer, Name und Studiengang gespeichert.
- Zu jedem Dozenten werden Personalnummer und Fachgebiet gespeichert.

---

## Aufgabe 4 – Bibliothek (mittel)

Eine Bibliothek verwaltet Bücher und Mitglieder.

- Jedes Buch besitzt eine ISBN, einen Titel und ein Erscheinungsjahr.
- Ein Autor kann mehrere Bücher schreiben.
- Ein Buch kann von mehreren Autoren geschrieben werden.
- Mitglieder können Bücher ausleihen.
- Für jede Ausleihe werden Ausleihdatum und Rückgabedatum gespeichert.
